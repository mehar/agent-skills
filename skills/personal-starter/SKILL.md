# Personal Starter

Create or update skills in this repository using this baseline:

1. Keep `name` in kebab-case and under 64 characters.
2. Write a clear `description` that explains what the skill does and when to trigger it.
3. Keep this file focused on workflow and decision points.
4. Place long reference material in `references/`.
5. Place reusable scripts in `scripts/` when deterministic execution is needed.
6. Place output templates and static files in `assets/`.

When cloning this starter into a new skill:

1. Rename folder and `name` field.
2. Replace description with specific trigger language.
3. Remove placeholder files not needed.
